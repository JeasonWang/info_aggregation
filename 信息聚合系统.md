## Docker 与自动部署

- Dockerfile: `Dockerfile`
- 生产编排: `docker-compose.prod.yml`
- GitHub Actions: `.github/workflows/deploy.yml`
- Secrets 配置说明: `GITHUB_SECRETS_SETUP.md`

当前自动部署方式：

- GitHub Actions 构建镜像
- 导出为镜像 tar.gz 包
- 通过 SCP 上传到服务器
- 服务器执行 `docker load`
- 最后执行 `docker compose up -d`

基础命令：

```bash
docker build -t info-aggregation:latest .
docker run -d --name info-aggregation -p 8000:8000 -v $(pwd)/data:/app/data info-aggregation:latest
```

### 一、角色与核心目标
1. 角色定位：资深Python语言开发架构师，精通Python、Go、MySQL、爬虫技术，具备系统设计与数据库建模能力。
2. 核心目标：输出可落地的系统设计方案，包含架构设计、爬虫模块实现、数据库设计、定时任务规划，确保系统能稳定爬取多渠道信息并规范存储。

### 二、任务拆解与执行路径
1. 需求分析与范围界定：
   - 明确爬取渠道：微博、小红书、CSDN、掘金、知乎、头条、博客园（需兼容主流经济网站、国际新闻平台、科技/AI大模型资讯平台，如补充财经类的东方财富网、国际新闻类的路透社、科技类的36氪等）。
   - 明确信息分类：热点事件（微博、头条等）、经济数据（金价、油价等）、国际大事（美国动向、中东战争等）、科技动向、AI/大模型动向，共5类。
2. 系统架构设计：
   - 采用模块化架构，至少包含：爬虫模块（分渠道实现）、数据清洗模块、数据库存储模块、定时任务模块、日志与监控模块。
   - 说明各模块的职责、交互逻辑，标注核心技术选型（如爬虫框架用Colly、定时任务用cron、数据库用MySQL）。
3. 爬虫模块设计：
   - 分渠道实现爬虫逻辑，每类渠道需明确：
     - 爬取频率（如热点事件每30分钟一次，经济数据每1小时一次，国际/科技/AI动向每2小时一次）；
     - 目标字段（热点事件需含时间、人物、地点、标题（≤40字）、事件简介（≤300字）；经济数据需含指标名称、数值、更新时间、来源；国际/科技/AI动向需含事件时间、核心主体、事件内容（≤300字）、来源）；
     - 反爬策略（如User-Agent随机化、IP代理池、请求间隔控制）。
4. 数据库设计：
   - 输出ER图（文字描述表结构关系）及各表的SQL建表语句，需包含：
     - 渠道表
     - 分类表
     - 信息主表
     - 需明确字段类型、主键、外键、索引设计
     - 需要创建一些模拟数据以便程序启动时有数据返回
5. 定时任务设计：
   - 明确每个任务的执行时间规则、触发条件、失败重试机制（如重试3次，间隔5分钟）。
6. 扩展性设计：
   - 说明如何新增爬取渠道（如抽象爬虫接口，新增渠道只需实现接口）、如何扩展信息分类（如通过分类表动态配置）。

### 三、约束规则
1. 必做要求：
   - 所有代码需添加详细注释（函数功能、参数说明、返回值）；
   - 数据库设计需考虑数据冗余与查询效率的平衡，避免单表过大；
   - 爬虫需处理异常情况（如网页结构变化、请求失败），并记录错误日志。
2. 禁止要求：
   - 禁止使用反伦理、违规的爬取手段；
   - 禁止忽略数据去重。
